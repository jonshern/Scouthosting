# Compass вЂ” mobile (parent app + team chat)

Scaffold for the Compass parent mobile app and the team chat
experience. Part of the Compass rebrand of Scouthosting.

> Source files are authored against the locked design system + the
> Compass JSON API. Most screens are wired to real endpoints; install
> + native build wiring (fonts, real photo uploads, EAS pipeline) is
> left as TODOs for the reviewer.

## Stack

- React Native via Expo SDK 51
- TypeScript (strict mode)
- React Navigation 6 (bottom tabs + native stack)
- react-native-svg for the compass-rose mark, photo placeholders, and
  the icon set
- expo-font for Newsreader (display) + Inter Tight (UI). System
  fallbacks render until fonts load.
- vitest for the design-token smoke test (no react-native runtime)

## Install + run

```bash
cd mobile
npm install
npx expo start
```

Press `i` for the iOS simulator or `a` for Android.

## Build (EAS)

`eas.json` is checked in with three profiles: `development`, `preview`,
and `production`. Before the first build:

```bash
npm install -g eas-cli
eas login                       # authenticate with your Expo account
eas init                        # links this directory to an EAS project
                                # (creates "extra.eas.projectId" in app.json)
```

Then:

```bash
eas build --profile development --platform ios       # simulator-friendly dev build
eas build --profile preview --platform android       # internal-distribution APK
eas build --profile production --platform all        # store-ready binaries
```

Signing credentials are managed by EAS вЂ” first run will prompt you
to either generate fresh certificates / keystores or upload your own.

### Submitting

The `submit.production` block in `eas.json` has placeholder values
(`REPLACE_WITH_*`). Fill those in (App Store Connect app id, Apple ID
email, Apple Team ID, Play service-account JSON path) before running:

```bash
eas submit --platform ios --latest
eas submit --platform android --latest
```

### Branding assets

`assets/icon.png` (1024Г—1024), `assets/adaptive-icon.png` (1024Г—1024
foreground for Android), `assets/splash.png` (1242Г—2436), and
`assets/favicon.png` (48Г—48) ship as **placeholder** PNGs generated by
`scripts/gen-placeholder-icons.mjs`. Replace each with branded artwork
before App Store / Play Store submission. The placeholders are valid
images so `eas build` works today; they're just plain forest-green
tiles with a white "C" mark.

### Backend pointer

The mobile client picks up the backend URL from
`EXPO_PUBLIC_COMPASS_BASE_URL` at build time (defaults to per-org
subdomains under `compass.app`). Set it in `.env` (gitignored) when
building against staging or a local tunnel:

```
EXPO_PUBLIC_COMPASS_BASE_URL=https://staging.compass.app
```

## Design system parity

- `src/theme/tokens.ts` mirrors the `bold` palette from
  `design/source/tokens.js` (Forest & Ember). The smoke test enforces
  this 1:1.
- The signature italic + chartreuse accent-fill display treatment
  appears on `HomeScreen` (greeting), `ThreadScreen` (hero line),
  `EventDetailScreen` (hero), and `PhotoPermissionsScreen` (headline).
- No emojis are baked into the codebase. Mock content from the design
  reference (e.g. emoji glyphs in chat) is rendered via short text
  glyphs ("HP", "T12", etc.) inside icon blocks.

## Run the tokens smoke test

From the worktree root:

```bash
npx vitest run mobile/tests/ --config mobile/vitest.config.ts
```

The test reads `design/source/tokens.js` (the canonical reference) and
`mobile/src/theme/tokens.ts` (the mobile mirror) without loading
react-native and asserts every Forest & Ember palette value matches.
There are 29 assertions covering name, base palette, and full
secondary spectrum.

## Open work

Things that need real-world inputs we can't synthesize:

- **Branded artwork** вЂ” replace the placeholder icon, adaptive-icon,
  splash, and favicon under `assets/` with real designs (re-run
  `scripts/gen-placeholder-icons.mjs` if you just want regen'd
  placeholders).
- **Fonts** вЂ” drop Newsreader + Inter Tight TTFs into `assets/fonts/`
  and re-introduce a `useFonts({ ... })` block in `App.tsx`. System
  fonts render in the meantime.
- **Auth providers** вЂ” Google + Apple SSO ship today. WebAuthn passkey
  enrollment + Microsoft SSO are still TODO.
- **Push notifications** вЂ” registration on launch works via
  `expo-notifications`. Real-device verification (APNs / FCM
  certificates) and badge counts are the remaining pieces.
- **Submission credentials** вЂ” fill in the `REPLACE_WITH_*` fields in
  `eas.json`'s `submit.production` block before running
  `eas submit`.
- **Universal-link verification files** вЂ” host the
  `apple-app-site-association` JSON at
  `https://compass.app/.well-known/apple-app-site-association` and
  `assetlinks.json` at
  `https://compass.app/.well-known/assetlinks.json`. The app config
  is ready (see `app.json` `ios.associatedDomains` and
  `android.intentFilters`); the server-side files belong to the apex
  rather than this repo.
